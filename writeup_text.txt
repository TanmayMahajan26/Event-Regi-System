Theory:
Modern web applications follow a three-tier architecture:
Frontend (Client-side) – User Interface 
Backend (Server-side) – Business Logic 
Database – Data Storage 
MongoDB integration with frontend means enabling the UI to:
Send data to the database 
Retrieve and display stored data 
Perform CRUD operations through APIs
CRUD) operations

2. Detailed Step-by-Step Flow
Step 1: User Interaction
User enters data in the frontend form (e.g., name, email, age) 
Clicks a button like “Add User” 

Step 2: Frontend Processing
Frontend captures input values 
Sends an HTTP request using: 
fetch() or axios 
👉 Example:
fetch('http://localhost:3000/users', {  method: 'POST',  body: JSON.stringify(user),  headers: { 'Content-Type': 'application/json' }});

Step 3: API Request Handling (Backend)
Request reaches Express server 
Express routes handle the request: 
/users → POST, GET, PUT, DELETE 

Step 4: Business Logic Layer
Backend validates data 
Applies logic (e.g., required fields, formatting) 

Step 5: Database Interaction
Backend uses Mongoose to interact with MongoDB: 
save() → Insert data 
find() → Retrieve data 
findByIdAndUpdate() → Update 
findByIdAndDelete() → Delete 

Step 6: MongoDB Execution
MongoDB performs operation 
Stores or fetches data from collections







Step 7: Response Sent Back
Backend sends response (JSON format) 
{  "message": "User added successfully"}

Step 8: Frontend Update
Frontend receives response 
Updates UI dynamically: 
Refresh list 
Show success message
